# Карта доказательства инвариантов

## Назначение

Этот документ связывает ключевые инварианты Procwright с владельцами в коде и проверками. Test/eval доказывает инвариант, но
не владеет им. Владелец должен быть value object, policy, resolver, state machine, validator или runtime component.

## Core invariants

| Инвариант | Владелец | Proof |
| --- | --- | --- |
| Пользователь выбирает сценарий, а не runtime flags. | `CommandService`, `ScenarioProfile`, scenario invocation builders. | `CommandServiceTest`, `ScenarioEntrypointIntegrationTest`, scenario docs, `context/scenario-contracts.md`. |
| Runtime получает validated execution/session plan. | `ExecutionPlanResolver`, `LaunchPlan`, `ExecutionPlan`, `SessionExecutionPlan`, `StreamExecutionPlan`. | `ExecutionPlanResolverTest`, integration tests for `run`, `interactive`, `lineSession`, `listen`. |
| Direct argv является безопасным default, shell mode явный. | `CommandSpec`, `CommandValidation`, `SystemShell`. | `CommandSpecTest`, `OneShotExecutionIntegrationTest`, security docs. |
| Environment inheritance является явной policy с compatibility default. | `EnvironmentPolicy`, command/session/stream invocation builders, resolver. | `CommandInvocationTest`, `SessionInvocationTest`, `StreamInvocationTest`, `OneShotExecutionIntegrationTest`, security docs. |
| One-shot output bounded by policy and reports truncation. | `CapturePolicy`, `CapturedOutput`, `ProcessKernel`, `CommandResult`. | `CapturedOutputTest`, `CommandResultTest`, `OneShotExecutionIntegrationTest`, `ProcessStressTest`. |
| Timeout завершает process tree через shutdown policy. | `RunOptions`, `ShutdownPolicy`, `ProcessLifecycle`, `ProcessKernel`, session runtime. | `OneShotExecutionIntegrationTest`, `ProcessStressTest`, `TestCliStressTest`. |
| Stdin ownership is explicit and safe. | `CommandInput`, `StdinPolicy`, command/session/stream runtimes. | `OneShotExecutionIntegrationTest`, `InteractiveSessionIntegrationTest`, `StreamScenarioIntegrationTest`. |
| Procwright-produced runtime failures share one generic catch boundary without losing scenario-specific details. | `ProcwrightException`, scenario-specific exception classes, optional integration exceptions. | `ProcwrightExceptionTest`, `IntegrationExceptionTest`, results/errors docs. |
| Session output ownership cannot be shared by incompatible readers. | `SessionOutputOwnership`, `DefaultSession`, scenario wrappers. | `SessionOutputOwnershipTest`, `ExpectIntegrationTest`, `LineSessionIntegrationTest`, streaming tests. |
| Raw session lifecycle is idempotent and observable. | `DefaultSession`, `SessionExit`, `SessionOptions`. | `InteractiveSessionIntegrationTest`, Kotlin await tests, diagnostics tests. |
| Line requests are serialized and cannot interleave responses. | `DefaultLineSession`, `ResponseDecoder`, line request lock/state. | `LineSessionIntegrationTest`, `PooledLineSessionIntegrationTest`, `ProcessStressTest`. |
| Protocol requests are serialized and adapter-owned without exposing raw leases. | `DefaultProtocolSession`, `ProtocolAdapter`, `ProtocolReader`, `ProtocolWriter`, protocol request lock/state. | `ProtocolSessionIntegrationTest.parallelRequestsAreSerialized`, `ScenarioEntrypointIntegrationTest.protocolSessionScenarioOpensTypedWorker`, protocol-session docs. |
| Readiness probes run after launch and before session or worker availability. | `ReadinessSupport`, `ScenarioRuntime`, `SessionInvocation`, `LineSessionInvocation`, `ProtocolSessionInvocation`. | `LineSessionIntegrationTest`, `ProtocolSessionIntegrationTest`, scenario contracts. |
| Strict charset decoding can report malformed output as typed failure. | `CharsetPolicy`, `ProcessKernel`, `DefaultLineSession`, `DefaultProtocolSession`, transcript buffers. | `CharsetPolicyTest`, `OneShotExecutionIntegrationTest.strictCharsetPolicyReportsDecodeErrorAsTypedFailure`, `LineSessionIntegrationTest.strictCharsetPolicyReportsDecodeErrorAsTypedFailure` (+ `multiByteCodepointSplitAcrossChunksDecodesCorrectly` для incremental decoder), `ProtocolSessionIntegrationTest.strictCharsetPolicyReportsMalformedOutput`, results/errors docs. |
| Protocol request/response size limits are separate from transcript retention. | `ProtocolSessionOptions`, `ProtocolRequestWriter`, `ProtocolResponseReader`, `ProtocolResponseBudget`. | `PolicyValueTest`, `ProtocolSessionIntegrationTest`. |
| Protocol failure taxonomy is stable and domain-mappable. | `ProtocolSessionException`, `PooledProtocolSessionException`, `ProtocolRuntimeFailures`, `CliAdapterError`. | `ProtocolSessionIntegrationTest`, `CliAdapterErrorTest`, results/errors docs. |
| Line transcripts and stream diagnostics are bounded. | `BoundedTranscriptBuffer`, `LineTranscript`, `StreamTranscript`, `ExpectOptions`, `LineSessionOptions`, `StreamOptions`. | `BoundedTranscriptBufferTest`, line/expect/stream integration tests. |
| Expect action values are redacted by default in transcripts and failure messages. | `ExpectOptions`, `ExpectTranscriptValues`, `DefaultExpect`. | `ExpectIntegrationTest`, security docs. |
| Terminal-required scenarios never silently fall back to pipes. | `TerminalPolicy`, `PtyProvider`, `ProcessTransport`. | `TerminalCapabilityBoundaryTest`, `PtyTransportIntegrationTest`, platform docs. |
| Streaming does not retain full output and propagates listener failures. | `StreamOptions`, `StreamInvocation`, `DefaultStreamSession`, `StreamRuntime`. | `StreamScenarioIntegrationTest`, diagnostics tests, stress tests. |
| Diagnostics do not change execution behavior. | `DiagnosticsOptions`, `DiagnosticEmitter`, async listener/sink delivery. | `DiagnosticsIntegrationTest`, `DiagnosticsOptionsTest`, diagnostics docs. |
| Public API surface does not grow accidentally and does not leak internal or external process-library types. | JPMS descriptors, exact public API baseline tests, external boundary check. | `PublicApiSurfaceTest`, `PublicIntegrationApiSurfaceTest`, `PublicKotlinApiSurfaceTest`, `PackageBoundaryTest`, `ExternalLibraryBoundaryTest`, `externalLibraryBoundaryCheck`. |
| Pooled line sessions reuse existing line-session runtime rather than creating a second process runtime. | `PooledLineSessionInvocationDefaults`, `DefaultPooledLineSession`, `SessionScenarioSupport`. | `PooledLineSessionInvocationDefaultsTest`, `PooledLineSessionIntegrationTest`, stress tests. |
| Pooled protocol sessions reuse existing protocol-session runtime and keep adapter state worker-owned. | `protocolSession(factory).pooled()`, serialized per-worker adapter factory, `PooledProtocolSessionInvocation`, `DefaultPooledProtocolSession`, `SessionScenarioSupport`. | `ProtocolSessionIntegrationTest.pooledProtocolCreatesOneAdapterPerWorker`, `pooledProtocolSerializesAdapterFactoryCalls`, `ScenarioEntrypointIntegrationTest.pooledProtocolScenarioBranchesAfterProtocolSessionChoice`, `TestCliStressTest.pooledProtocolSessionRetiresTimedOutWorkersAndKeepsServingRequests`. |
| Pool hooks are bounded and pool metrics separate acquire wait from request duration. | `PooledLineSessionOptions`, `PooledProtocolSessionOptions`, `DefaultPooledLineSession`, `DefaultPooledProtocolSession`, `WorkerHookSupport`. | `PooledLineSessionIntegrationTest`, `ProtocolSessionIntegrationTest`, `PolicyValueTest`. |
| Pool metrics snapshots cannot represent impossible state. | `PooledLineSessionMetrics`, `PooledProtocolSessionMetrics`. | `PolicyValueTest`, pool integration tests. |
| Scenario presets do not create runners or bypass builders/resolver. | `ScenarioPresets`. | `ScenarioPresetsTest`, scenario presets docs, ADR-0008. |
| Kotlin ergonomics do not add Kotlin dependency to Java core. | Gradle module boundaries, `:procwright-kotlin`. | `PublicKotlinApiSurfaceTest`, `externalLibraryBoundaryCheck`, dependency review. |
| CLI integrations do not pull MCP SDK or external process libraries into core. | `:procwright-integrations`, module descriptor, dependency review. | `IntegrationModuleDescriptorTest`, `CommandBackedToolTest`, `ExternalLibraryBoundaryTest`. |
| JSON/framing helpers reject malformed or oversized protocol data. | `JsonCodec`, `JsonLines`, `ContentLengthJsonFrames`, `ProtocolAdapters`, `IntegrationProtocolException`. | `JsonCodecTest`, `ContentLengthJsonFramesTest`, `JsonLineSessionTest`, `ProtocolAdaptersTest`. |
| Test CLI can model unstable real-world process behavior. | `:procwright-test-cli` scenario registry and scenario implementations. | `:procwright-test-cli:check`, `TestCliStressTest`, `context/evals/test-cli-simulator.md`. |
| `closeStdin()` возвращается сразу даже когда конкурентный писатель заблокирован полным stdin pipe; заблокированная запись завершается typed failure после разрыва pipe. | `DefaultSession` state machine + фоновое закрытие raw stdin. | `InteractiveSessionIntegrationTest.closeStdinReturnsPromptlyWhileWriterIsBlockedOnFullPipe`, `sendAfterCloseStdinFails`. |
| Лимиты длины строки применяются к содержимому строки независимо от LF/CRLF терминатора. | `ProtocolResponseReader.readLine`, `DefaultLineSession.publishLines`. | `ProtocolResponseReaderTest`, `LineSessionIntegrationTest` (exact-length LF/CRLF cases). |
| Непрочитанный stderr не убивает protocol session: stdout backlog строгий, stderr bounded drop-oldest и остается читаемым до лимита. | `DefaultProtocolSession`, `ProtocolOutputQueue.OverflowPolicy`. | `ProtocolSessionIntegrationTest.unreadChattyStderrDoesNotKillLongLivedProtocolSession`, `stderrStaysReadableThroughBoundedQueue`, `stdoutBacklogOverflowIsVisibleWhenAdapterReadsOtherStream`. |
| Потомки, пережившие выход процесса и удерживающие output pipe, уничтожаются в failure cleanup, а сообщение об ошибке объясняет причину. | `ProcessLifecycle.waitFor` (descendants snapshot), `ProcessLifecycle.forceStop(knownDescendants)`, `ProcessKernel`. | `OneShotExecutionIntegrationTest.orphanedDescendantHoldingOutputPipeIsKilledAndFailureExplainsCause`. |
| Diagnostics события одного run доставляются каждому получателю последовательно в порядке emit, не блокируя runtime. | `DiagnosticEmitter.SerialDelivery`. | `DiagnosticEmitterTest`, `DiagnosticsIntegrationTest.runDiagnosticEventsArriveInLifecycleOrder`. |
| Первый terminal failure сессии сохраняется и surface-ится последующими `request()` вместо generic CLOSED; смерть процесса при записи классифицируется как `PROCESS_EXITED`. | Terminal-failure state в `DefaultLineSession`/`DefaultProtocolSession`, `ProcessExitedException`. | `LineSessionIntegrationTest.requestAfterStdoutBacklogOverflowReportsOverflowReason`, `requestAgainstExitedProcessReportsProcessExited`, протокольные аналоги в `ProtocolSessionIntegrationTest`. |
| Recovery частичного output после timeout force-stop опирается на состояние shutdown, а не на текст сообщения JDK. | `ProcessKernel.shutdownOutput`. | `OneShotExecutionIntegrationTest` timeout/partial-output tests. |
| Expect по умолчанию читает в charset сессии; явный `ExpectOptions` charset имеет приоритет. | `ExpectOptions.charsetFor`, `DefaultExpect`. | `ExpectIntegrationTest.expectFollowsSessionCharsetByDefault`, `explicitExpectCharsetTakesPrecedenceOverSessionCharset`. |
| `Duration.ZERO` отключает run timeout (как и idle/stream timeout): процесс ожидается до собственного выхода, shutdown policy продолжает действовать на failure paths; отрицательный timeout отклоняется рано. | `ProcessLifecycle.waitFor` (единственный владелец семантики «ноль = без deadline»), валидация в `RunOptions`/`CommandInvocation.Builder`/`ExecutionPlan`. | `OneShotExecutionIntegrationTest.zeroTimeoutDisablesRunTimeoutAndAwaitsCompletion`, `negativeTimeoutIsRejectedBeforeLaunch`, `ExecutionPlanResolverTest.resolvesZeroTimeoutAsDisabledRunTimeout`, `RunOptionsTest`. |
| Discard/file capture перенаправляет output на уровне ОС без pump threads, не меняя timeout supervision и process-tree cleanup; в `CommandResult` пустые stdout/stderr и `false` truncation flags. Несовместимые комбинации (merged-файл при `SEPARATE`, два файла при `MERGED`, совпадающие или пустые пути) отклоняются до запуска процесса. | `CapturePolicy.ToPath` (value object), `ExecutionPlan` (совместимость с `OutputMode`), `ProcessKernel.stdioConfig`, `StdioConfig`, `ProcessLifecycle.start`. | `PolicyValueTest` (toPath/discard), `ExecutionPlanResolverTest.rejectsMergedFileCaptureWithSeparateOutputMode`, `rejectsTwoFileCaptureWithMergedOutputMode`, `OneShotExecutionIntegrationTest.fileCaptureWritesLargeOutputWithEmptyResultStreams`, `discardCaptureDropsOutputWithoutFailing`, `mergedSingleFileCaptureReceivesBothStreams`, `timeoutWithFileCaptureStillStopsProcessAndReportsTimedOut`. |
| `CommandInput.fromPath` стримит stdin из файла через OS-redirect, не загружая его в память Procwright; отсутствующий файл дает typed `CommandExecutionException` с reason `LAUNCH_FAILED` до старта writer-задач. | `CommandInput` (value object), `ProcessKernel.stdinRedirect`, `StdinPolicy`. | `CommandInputTest` (fromPath cases), `OneShotExecutionIntegrationTest.stdinFromPathStreamsLargeFileWithoutBufferingInMemory`, `stdinFromMissingFileFailsWithTypedLaunchFailure`. |
| `ExpectMatch` возвращает текст совпадения, capture groups и `before` из match buffer; результат — живой output без редактирования (в отличие от transcript), redaction policy транскриптов не ослабляется, таймаут остается typed `ExpectException`. | `ExpectMatch` (value object), `DefaultExpect.expectTextMatch`/`expectRegexMatch` (единственный matching loop, fluent-методы делегируют). | `ExpectMatchTest`, `ExpectIntegrationTest.literalMatchResultReturnsMatchedTextEmptyGroupsAndBeforeText`, `regexMatchResultExtractsValueThroughCaptureGroups`, `consecutiveMatchResultsReportBeforeTextBetweenMatches`, `matchResultTimeoutIsTypedExpectException`. |
| Прерывание caller-потока, заблокированного в `run()` или `lineSession request()`, дает typed failure, восстанавливает interrupt status и останавливает процесс. | `ProcessLifecycle.waitFor` (re-interrupt + tree cleanup), `DefaultLineSession.writeLine`/`ResponseReader.readLine`, `ProcessKernel.await`. | `OneShotExecutionIntegrationTest.callerInterruptDuringRunIsTypedFailureAndRestoresInterruptStatus`, `LineSessionIntegrationTest.callerInterruptDuringRequestIsTypedFailureAndRestoresInterruptStatus`. |
| Процесс, игнорирующий interrupt signal (SIGTERM), принудительно убивается после interrupt grace; `run` сообщает `timedOut`, cleanup остается bounded. | `ProcessLifecycle.stop` (escalation branch, общий для run и session shutdown), `ShutdownPolicy`. | `OneShotExecutionIntegrationTest.shutdownEscalationForceKillsProcessThatSurvivesInterruptSignal` (POSIX). |
| LF/CRLF терминаторы декодируются без ручной нормализации на line/expect уровне; одинокий `\r` — содержимое строки, а не терминатор; raw capture `run` сохраняет стиль newline без нормализации. | `DefaultLineSession.publishLines`, `DefaultExpect` (raw match buffer), `CapturedOutput`. | `LineSessionIntegrationTest.crlfTerminatedResponsesDecodeWithoutTerminators`, `mixedLineTerminatorsWithinOneSessionDecodeWithoutManualNormalization`, `carriageReturnWithoutLineFeedIsContentNotTerminator`, `ExpectIntegrationTest.expectMatchesCrlfTerminatedOutputWithoutNormalization`, `OneShotExecutionIntegrationTest.capturedOutputPreservesEmittedNewlineStyleWithoutNormalization`. |
| Protocol и pooled protocol сценарии испускают lifecycle diagnostics со scenario name, общим `runId` на process lifecycle и без сырых значений. | `ScenarioRuntime` (scenario names `protocolSession`/`pooledProtocol`), `DiagnosticEmitter`, `DiagnosticEvent.runId`. | `DiagnosticsIntegrationTest.protocolSessionEmitsLifecycleEventsWithSharedRunId`, `pooledProtocolWorkersEmitLifecycleEventsWithSharedRunId`. |

## Release proof gates

- Fast contracts: `./gradlew quickCheck`.
- Scenario behavior: `./gradlew scenarioCheck`.
- Regression and bounded stress: `./gradlew regressionCheck`.
- Public docs and Java API docs: `./gradlew publicDocsCheck`.
- Complete local gate: `./gradlew releaseCandidateCheck`.

## Maintenance rule

Любой новый public API или behavior должен добавлять строку в эту карту либо явно расширять существующий инвариант.
Если владелец инварианта неочевиден, изменение не готово к merge.
