# ADR-0004: Java 25 baseline

## Статус

Принято как решение о development baseline. Релизная совместимость уточнена и заменена
[ADR-0020](ADR-0020-java-release-variants.md).

## Контекст

Первоначально clean rewrite выбрал Java 21 как зрелый LTS baseline. Это решение оптимизировало библиотеку под более
широкую практическую применимость.

Приоритет проекта изменен: iCLI в первую очередь должен быть образцом высококлассного современного кода и архитектуры,
а не максимально совместимым production-артефактом. Поэтому baseline должен позволять использовать наиболее сильные
доступные сейчас возможности Java, если они помогают сделать API, инварианты и runtime проще, выразительнее и надежнее.

Локальная среда компилирует проект JDK 26, но артефакты библиотеки должны иметь явный target, чтобы случайно не
поднимать baseline без нового решения.

## Решение

Целевой baseline для core library: Java 25.

Сборка должна компилировать production и test classes с `--release 25`. В тестах есть проверка class file major version,
чтобы baseline был механически зафиксирован, а не жил только в документации.

## Последствия

Плюсы:

- можно использовать современные возможности языка и платформы без оглядки на Java 21 compatibility;
- проект лучше соответствует цели показать максимально качественный современный JVM-код;
- меньше давления на преждевременные compatibility abstractions;
- API и runtime-решения можно проектировать смелее.

Минусы:

- аудитория потенциальных пользователей уже, чем при Java 21;
- tooling и IDE пользователей должны поддерживать Java 25;
- каждый переход на baseline выше Java 25 все равно должен проходить через отдельный ADR и проверку build target.
