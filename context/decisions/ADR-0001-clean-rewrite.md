# ADR-0001: чистая ветка для переписывания

## Статус

Accepted.

## Контекст

Старая версия iCLI содержит полезные идеи, но ее кодовая и документная архитектура слишком тяжелая для дальнейшего
развития. Инкрементальный рефакторинг будет постоянно тащить старые публичные классы и старую терминологию.

Пользователь попросил новую чистую ветку с новым русскоязычным контекстом. Код и будущие public API должны быть
на английском.

## Решение

Создать orphan-ветку `rewrite/clean-context` без старых файлов и перенести только компактный контекст:

- архитектурную рамку MVP;
- идеи пользовательского API;
- уроки старой реализации;
- легкую модель разработки с агентами;
- eval/quality ориентиры.

Старую реализацию использовать как reference через историю `main`, а не как рабочее дерево.

## Последствия

Плюсы:

- нет старого кода, который будущие агенты будут механически продолжать;
- контекст стал меньше и написан на языке пользователя;
- API-идеи сохранены отдельно от старой реализации;
- первый MVP можно строить вокруг runtime correctness, а не вокруг compatibility.

Минусы:

- историю старых файлов нужно смотреть через Git;
- старые scripts/build/test infrastructure нужно будет создать заново;
- потребуется заново принять решения по Java baseline, Gradle structure и PTY packaging.

## Альтернативы

### Рефакторинг в старой ветке

Отклонено: слишком много старой структуры и публичных концепций будет влиять на новые решения.

### Сохранить весь `context/`

Отклонено: старый контекст слишком тяжел как ежедневная инструкция. Если исторические детали понадобятся, их нужно
читать из Git history, а не хранить в текущем `context/`.

### Удалить все кроме одного README

Отклонено: агентам нужен минимальный, но структурированный набор источников правды.
